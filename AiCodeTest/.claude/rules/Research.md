# Research (조사 단계)

## 목적

구현 전에 요구사항, 기존 코드, 관련 도메인을 충분히 파악하여 잘못된 방향으로 개발하는 것을 방지한다.

## 수행 절차

### 1. 요구사항 파악
- 사용자의 요청을 명확히 이해한다
- 모호한 부분은 구현 전에 반드시 질문하여 확인한다
- 기능의 범위(scope)를 명확히 정의한다

### 2. 기존 코드 분석
- 관련 도메인의 기존 코드를 먼저 읽는다
- 유사한 기능이 이미 구현되어 있는지 확인한다 (중복 구현 방지)
- 공통 모듈(`modules/common/`, `composables/`, `components/`)에 재사용 가능한 코드가 있는지 확인한다
- 기존 패턴(Controller/Service/Mapper, composable 구조)을 파악한다

### 3. 영향 범위 파악
- 변경 시 영향을 받는 파일/모듈 목록을 파악한다
- DB 스키마 변경이 필요한지 확인한다
- API 계약(요청/응답 형식)이 변경되는지 확인한다
- 프론트엔드-백엔드 인터페이스 변경 여부를 파악한다

### 4. 기술 제약 확인
- 사용 중인 기술 스택의 제약사항을 확인한다
  - 백엔드: Java 21, Spring Boot, MyBatis (XML 기반), MSSQL
  - 프론트엔드: Nuxt 3, Vue 3, TypeScript
- 기존 의존성으로 해결 가능한지, 새 라이브러리가 필요한지 판단한다

## 산출물

- 요구사항 요약 (무엇을 구현해야 하는가)
- 참조할 기존 코드 목록
- 영향 받는 파일/모듈 목록
- 기술적 제약 또는 주의사항
