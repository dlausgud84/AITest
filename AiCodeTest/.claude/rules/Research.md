# Research (조사 단계)

## 목적

1. 구현 전에 요구사항, 기존 코드, 관련 도메인을 충분히 파악하여 잘못된 방향으로 개발하는 것을 방지한다.
2. 멀티모듈 의존성 파악 (Backend): 현재 백엔드가 apps, modules/common, modules/auth, modules/menu 등의 멀티모듈로 구성되어 있습니다. 신규 기능이 기존 모듈에 속하는지, 혹은 새로운 모듈로 분리/추가해야 하는지 아키텍처 관점의 조사가 필요합니다.

3. 공통 UI/기능 자산 조사 (Frontend): Nuxt 3 환경에서 기존에 만들어둔 공통 컴포넌트(common-components-guide.md), 레이아웃, 공통 유틸리티(Composables), Pinia Store 상태를 재사용할 수 있는지 중복 개발 방지 차원에서 확인해야 합니다.

4. DB 프로시저 및 관계 파악 (Database): MSSQL과 MyBatis를 사용 중이므로, 변경하려는 테이블의 외래키(FK) 참조, 인덱스, 엮여있는 트리거/프로시저가 없는지 데이터베이스 영향도를 조사해야 합니다.
## 수행 절차

### 1. 요구사항 파악
- 사용자의 요청을 명확히 이해한다
- 모호한 부분은 구현 전에 반드시 질문하여 확인한다
- 기능의 범위(scope)를 명확히 정의한다

### 2. 기존 코드 분석
- 관련 도메인의 기존 코드를 먼저 읽는다
- 유사한 기능이 이미 구현되어 있는지 확인한다 (중복 구현 방지)
- 공통 모듈(`modules/common/`, `composables/`, `components/`)에 재사용 가능한 코드가 있는지 확인한다
- 기존 패턴(Controller/Service/Mapper, composable 구조)을 파악한다

### 3. 영향 범위 파악
- 변경 시 영향을 받는 파일/모듈 목록을 파악한다
- DB 스키마 변경이 필요한지 확인한다
- API 계약(요청/응답 형식)이 변경되는지 확인한다
- 프론트엔드-백엔드 인터페이스 변경 여부를 파악한다

### 4. 기술 제약 확인
- 사용 중인 기술 스택의 제약사항을 확인한다
  - 백엔드: Java 21, Spring Boot, MyBatis (XML 기반), MSSQL
  - 프론트엔드: Nuxt 3, Vue 3, TypeScript
- 기존 의존성으로 해결 가능한지, 새 라이브러리가 필요한지 판단한다

## 산출물

- 요구사항 요약 (무엇을 구현해야 하는가)
- 참조할 기존 코드 목록
- 영향 받는 파일/모듈 목록
- 기술적 제약 또는 주의사항
