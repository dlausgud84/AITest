# 1. 조사 (Research)

## 🎯 목표
요구사항을 분석하고, 연관된 기존 시스템 로직, 데이터베이스 구조 및 인프라 설정 상태를 파악합니다.

## ✅ 체크리스트

### 1. 요구사항 및 API 스펙 파악
- [ ] 기획/비즈니스 요구사항 분석 및 엣지 케이스(예외 상황) 도출
- [ ] 프론트엔드와 협의할 API 스펙(Request/Response, URI 명명 규칙) 초안 확인
- [ ] **[💡추가]** 페이징, 정렬, 검색 조건 등 목록 조회 시 필요한 파라미터 요구사항 확인

### 2. 보안 및 권한 분석 **[💡신규 섹션]**
- [ ] 해당 기능/API의 접근 권한(Role) 수준 파악 (예: 비회원, 일반회원, 관리자 등)
- [ ] 민감 정보(개인정보, 비밀번호 등) 포함 여부 및 암호화/마스킹 처리 요구사항 확인
- [ ] **[🔒보안추가]** SQL Injection 위험성 평가 (동적 쿼리 사용 시 Prepared Statement 필수)
- [ ] **[🔒보안추가]** XSS(Cross-Site Scripting) 방지 요구사항 (프론트엔드 출력 시 HTML 이스케이프)
- [ ] **[🔒보안추가]** CSRF(Cross-Site Request Forgery) 토큰 필요 여부 확인
- [ ] **[🔒보안추가]** 민감 데이터 로깅 금지 및 감사 로그(Audit Log) 요구사항 확인

### 3. 기존 코드 및 DB 분석
- [ ] `com.dit` 패키지 내 연관 도메인 및 레거시 코드 영향도 파악
- [ ] **[💡추가]** 재사용 가능한 기존 MyBatis `Mapper.xml` 쿼리 및 `ResultMap` 존재 여부 확인 (중복 개발 방지)
- [ ] MSSQL 관련 테이블 구조(DDL), 데이터 볼륨, 기존 관계(FK) 분석
- [ ] **[💡추가]** 연관된 MSSQL 저장 프로시저(Stored Procedure), 트리거(Trigger), 함수 존재 여부 파악

### 4. 외부 인터페이스 분석 **[💡신규 섹션]**
- [ ] 타 시스템 또는 외부 API(결제, 알림톡, 메일 발송 등) 연동 필요 여부 확인
- [ ] 연동 시 필요한 인증 키(API Key 등) 발급 상태 확인
